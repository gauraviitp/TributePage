Two Sum

Given an array of integers nums and an integer target, return the two numbers such that they add up to target.

You may assume that each input would have exactly one solution, and you may not use the same element twice.

You can return the answer in any order.

Example 1:

Input: nums = [2,7,11,15], target = 9
Output: [2,7]
Explanation: Because nums[0] + nums[1] == 9, we return [2, 7].

Example 2:

Input: nums = [3,2,4], target = 6
Output: [2,4]

Example 3:

Input: nums = [3,3], target = 6
Output: [3,3]
  • Only one valid answer exists.

Solution

Intuition

       x+y=target  ⟹  y=target−x \ \ \ \ \ \ \ x + y=target\\ \implies y = target-x        x+y=target⟹y=target−x

In this problem, we need to find out the two numbers x and y which sum up to the given target number. Thus, a number x forms a solution if another number y = target - x is also present in the list.

So, there are two tasks that need to be performed:

  1. Consider a number x in the list as a probable solution, and then
  2. find out if target - x also exists in the list.

If both the steps above are success, we have our answer.

Algorithm
We can come up with different approaches to perform the tasks described above.

We can fix the number x and then we can search the number y = target - x in the list. In this case, it effectively becomes a search problem. We could try linear search or a binary search (Not here that for binary search, we would also need to sort the list first).

Linear Search Approach

class Solution:
    def twoSum(self, nums: List[int], target: int) -> List[int]:
        n = len(nums)
        
        for i, x in enumerate(nums): 			#......... (1)
            y = target - x

            for j in range(i+1, n): 			#......... (2)
                if nums[j] == y:
                    return [x, y]

Runtime complexity
The outer loop (1) iterates through all the numbers in the list. And within each iteration, we execute loop (2) which also iterates through list starting at the next number.

You can visualise the runtime as follows:
for i=0, 1<=j<n  ⟹  n−11<=j<n \implies n-11<=j<n⟹n−1
for i=1, 2<=j<n  ⟹  n−22<=j<n \implies n-22<=j<n⟹n−2
and so on,
for i=n-2, n−1<=j<n  ⟹  1n-1<=j<n \implies 1n−1<=j<n⟹1
for i=n-1, n<=j<n  ⟹  0n<=j<n \implies 0n<=j<n⟹0

Summing over all i, we get n−1+n−2+n−3+...+3+2+1=n∗(n−1)/2≈n2n-1 + n-2 + n-3 + ...+3+2+1=n*(n-1)/2 \approx n^{2}n−1+n−2+n−3+...+3+2+1=n∗(n−1)/2≈n2

O(n2)O(n^{2})O(n2)

Binary Search Approach

Once you have sorted the input list, we can perform the two tasks described above as follows:

  1. Fix the number x, and then
  2. binary search y = target - x in the list

bisect(arr, x, lo=pos) returns the first position of the element x in the array arr starting at index pos.

class Solution:
    def twoSum(self, nums: List[int], target: int) -> List[int]:
        n = len(nums)
        nums.sort() 					# sort the list ........(1)
        
        for i, x in enumerate(nums):	# ......................(2)
            y = target - x

            j = bisect.bisect(nums[i+1:], y)		# ..............(3)
            if 0 <= j < n:
                return [x, y]

Runtime complexity
O(n log(n))O(n \ log(n))O(n log(n))

Set approach

We could also use a map or a dictionary to search for the number y.
In this approach, after we have visited a number x, we add it in the set of complements so that we can search it later in O(1)O(1)O(1)

class Solution:
    def twoSum(self, nums: List[int], target: int) -> List[int]:
        complements = set()
        
        for i, x in enumerate(nums):
            y = target - x
            
            if y in complements:
                return [y, x]
            
            complements.add(x)

Runtime complexity
O(n)O(n)O(n)
